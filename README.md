## ✨ Возможности

### 📢 Управление каналами
- **Интерактивное добавление канала**: Выберите канал прямо через интерфейс Telegram (кнопка «Поделиться каналом»). Бот автоматически проверит права администратора.
- **Безлимитное количество каналов**: Подключайте любое количество каналов.
- **Разделение по пользователям**: Каждый пользователь видит и управляет только своими каналами.

---

### ✍️ Конструктор постов
- **Поддерживаемые форматы контента**:
  - 📝 Текст (с форматированием)
  - 🖼️ Фото (с подписью)
  - 🎬 Видео (с подписью)
  - 📎 Документ / файл
  - 🎵 Аудио
  - 🎙️ Голосовое сообщение
  - 🎞️ GIF-анимация
  - 🎭 Стикер

- **Умное обновление контента**: Если вы сначала отправили фото, а потом решили добавить текст — бот установит его как подпись к фото, не удаляя медиафайл.

---

### 🎨 Форматирование текста

Бот поддерживает четыре режима разметки:

| Режим | Описание |
|-------|----------|
| **Авто Telegram** | Сохраняет разметку «как есть» из вашего приложения: **жирный**, _курсив_, `код`, ~~зачёркнутый~~, кастомные эмодзи Premium |
| **HTML** | Ручные HTML-теги: `<b>`, `<i>`, `<u>`, `<s>`, `<code>`, `<a href="...">` |
| **Markdown** | Устаревший Markdown: `*жирный*`, `_курсив_`, `` `код` `` |
| **MarkdownV2** | Расширенный Markdown с экранированием спецсимволов |

---

### 🔗 URL-кнопки под постом

Добавляйте кнопки-ссылки под публикацией:

**Формат ввода:**
```
Текст кнопки - https://example.com
Кнопка 1 - https://site.com | Кнопка 2 - https://other.com - red
```

- Несколько кнопок в один ряд разделяются символом `|`
- Каждая строка — новый ряд кнопок

**🎨 Цвет кнопок** (поддерживает Telegram Bot API 9.4+):

| Параметр | Псевдоним | Цвет |
|----------|-----------|------|
| `primary` | `blue` | 🔵 Синяя (по умолчанию) |
| `success` | `green` | 🟢 Зелёная |
| `danger` | `red` | 🔴 Красная |

Пример: `Купить - https://shop.com - green`

---

### 👍 Кнопки реакций-голосований

Добавляйте интерактивные кнопки-реакции под постом:

- Вводите до **8 эмодзи** через пробел или запятую: `👍 👎 🔥 ❤️`
- Бот считает голоса: каждый читатель может поставить только **один голос**
- Результат отображается рядом с эмодзи: `👍 12  👎 3`

---

### 📅 Публикация

- **🚀 Немедленная публикация** — пост отправляется сразу
- **⏰ Отложенная публикация** — введите дату и время в формате `ДД.ММ.ГГГГ ЧЧ:ММ` (МСК, UTC+3)  
  _Пример: `15.08.2025 14:30`_
- Встроенный планировщик на базе **APScheduler** — работает в фоне, не требует внешних очередей

---

### 📜 История постов

- Просматривайте все опубликованные посты по каждому каналу
- **Редактирование опубликованных постов**:
  - Изменение текста/подписи прямо на канале (через Telegram Bot API)
  - Изменение URL-кнопок под любым опубликованным сообщением
  - Поддержка **пересланных сообщений**: перешлите боту любое сообщение со своего канала — он распознает его и предложит меню редактирования

## 🛠️ Установка и запуск

### Требования
- Python **3.9+**
- Токен бота от [@BotFather](https://t.me/BotFather)
- Права администратора в целевом Telegram-канале

### 1. Клонировать репозиторий

```bash
git clone https://github.com/novirx-tg/postingpost.git
cd postingpost
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Настроить конфигурацию

Переименуйте `.env.template` в `.env` и заполните:

```env
BOT_TOKEN=1234567890:ABCdefGhIJKlmNoPQRsTUVwxyZ
```

### 4. Запустить бота

```bash
python main.py
```

## 📁 Структура проекта

```
postingpost/
├── main.py                    # Точка входа, инициализация бота
├── config.py                  # Загрузка конфигурации из .env
├── database.py                # Работа с SQLite (каналы, посты, реакции)
├── requirements.txt
├── .env.template              # Шаблон конфигурации
├── handlers/
│   ├── start.py               # /start, главное меню
│   ├── channels.py            # Управление каналами
│   ├── post_creator.py        # Конструктор постов (FSM)
│   ├── history.py             # История и редактирование постов
│   └── scheduler.py           # Управление запланированными постами
└── services/
    ├── post_service.py        # Логика отправки постов, сборка клавиатур
    └── scheduler_service.py   # Планировщик APScheduler
```

## 🔒 Безопасность

- Файл `.env` с токеном **не загружается** в репозиторий (добавлен в `.gitignore`)
- База данных `posting_bot.db` также исключена из коммитов
- Каждый пользователь видит только свои каналы и посты
